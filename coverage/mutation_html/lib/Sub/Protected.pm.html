  104: =head1 DESCRIPTION
  105: 
  106: Enforces Java/C++-style "protected" access at runtime: a subroutine
  107: decorated with C<:Protected> (or named in C<use Sub::Protected qw(...)>)
  108: may only be called from within its defining package or from a subclass of
  109: that package.  Any other caller causes a C<Carp::croak> with a descriptive
  110: message.
  111: 
  112: =head2 Two usage forms
  113: 
  114: =over 4
  115: 
  116: =item Attribute form (preferred)
  117: 
  118:     sub _helper :Protected { ... }
  119: 
  120: The C<:Protected> attribute is registered in C<UNIVERSAL> via
  121: L<Attribute::Handlers> when C<Sub::Protected> is loaded, so every package
  122: has access to it without any further C<use> or inheritance.  The sub is
  123: wrapped at C<CHECK> time.  This form is preferred because the protection
  124: declaration sits next to the definition and wrapping happens at compile time
  125: (making pre-wrap raw-coderef captures impossible).
  126: 
  127: =item Declarative form
  128: 
  129:     use Sub::Protected qw(_helper _other);
  130: 
  131: Each named sub is looked up in the caller's stash and wrapped at C<CHECK>
  132: time (or immediately if the module is loaded at runtime via C<require>).
  133: All named subs must be defined before C<CHECK> fires -- i.e. they must be
  134: compile-time named subs in the same file, not generated at runtime.
  135: 
  136: =back
  137: 
  138: =head2 Bypass for testing
  139: 
  140: Either condition alone (OR logic) disables all access checks:
  141: 
  142: =over 4
  143: 
  144: =item * C<$Sub::Protected::BYPASS> set to a true value.  Use C<local> in tests.
  145: 
  146: =item * C<$ENV{HARNESS_ACTIVE}> set (the convention used by L<Test::Harness>/prove).
  147: 
  148: =back
  149: 
  150: C<$Sub::Protected::BYPASS> is the recommended form for new test code;
  151: it is explicit and does not depend on the test runner.
  152: C<HARNESS_ACTIVE> is a zero-config convenience.
  153: 
  154: The HARNESS_ACTIVE bypass can be disabled by setting:
  155: 
  156:     $Sub::Protected::config{harness_bypass} = 0;

  468: =head1 KNOWN LIMITATIONS
  469: 
  470: =over 4
  471: 
  472: =item Runtime-only
  473: 
  474: Checks are runtime only; there is no compile-time enforcement.
  475: 
  476: =item Raw coderef bypass
  477: 
  478: A raw code reference obtained B<before> wrapping (via C<can()> or direct
  479: C<\&Foo::_helper>) bypasses the check.  The attribute form prevents this
  480: because wrapping happens at compile time.
  481: 
  482: =item Moo/Moose method modifiers
  483: 
  484: Method modifiers applied after Sub::Protected has wrapped a sub will wrap
  485: the wrapper.  Apply Sub::Protected last, or use the declarative form in a
  486: C<CHECK> block after the class is fully built.
  487: 
  488: =item UNIVERSAL namespace pollution
  489: 
  490: The C<:Protected> attribute is installed in C<UNIVERSAL>, which is
  491: intentional (any package can use it after a single C<use>), but it does
  492: introduce C<UNIVERSAL::Protected> into the global namespace.
  493: 
  494: =item Thread safety
  495: 
  496: C<@_pending> and C<$BYPASS> are unguarded package globals.  Do not use
  497: concurrent C<use Sub::Protected qw(...)> calls across threads.
  498: 
  499: =back
